Motivação

Para a atividade do TDE, agendada para a semana que vem, é necessário o uso das funções sha1(), substr() e base64_encode.

Todas essas funções são muito simples e atuam sobre dados no formato string, embora sha1() e base64_encode() também possa atuar sobre um conjunto de bytes.

URLs Curtas

Um sistema de URLs curtas deve transformar uma URL para uma representação curta dela. No nosso caso, de 6 caracteres (letras maiúsculas e minúsculas e números).

Hash

Uma função hash é uma função de espalhamento. É um tipo de função de criptografia ("escrever escondido") de uma via. Isto é, um hash não pode ser facilmente desencriptado.

O algoritmo SHA-1 (Security Hash Algorithm) não é mais considerado seguro para criptografia. Atualmente, utiliza-se Bcrypt2 e suas variantes.

Antigamente, utiliza-se o algoritmo MD5-SUM para realizar esse processo

Hash

Uma das características de funções de hash é gerar valores muito distintos para chaves parecidas. Vamos testar no console:

1
2
3
4
php > print(sha1('alunos'));
258311a99c516651fafc071f3149ccbcf29bbc33 
php > print(sha1('alunas'));
713021c89e4b0138b7944d7b2bc63134dc606fe0

Note que as palavras diferem em apenas uma letra, mas o hash é distinto.

Base64_encode

O hash é o primeiro passo para se criar uma URL curta. O problema é que ele gera um valor muito longo, de 20 bytes, 40 caracteres.

Esse valor está em hexadecimal (0-9a-f).

Podemos utilizar uma outra representação, como base64, que utiliza números, letras maiúsculas e minúsculas para representar 64 valores por caracter.

A função base64_encode faz isso.

Tamanho da String

Nossa string ainda está muito grande, na verdade, maior do que antes. Para isso, podemos utilizar a função substr() para cortar um pedaço da string. Poderíamos cortar qualquer parte, mas é conveniente cortarmos a partir do começo:

1
2
3
4
php > $alunas_base64 = 'NzEzMDIxYzg5ZTRiMDEzOGI3OTQ0Z
DdiMmJjNjMxMzRkYzYwNmZlMA==' ;
php > print(substr($alunas_base64, 0, 6));
NzEzMD

O primeiro parâmetro é a string, o segundo é a posição inicial e o terceiro é o tamanho da string.

Processo

Esse processo encadeado:

sha1 -> base64_encode -> substr

permite a criação de URLs curtas, com baixa chance de colisão (mesma URL curta para duas URLs distintas).
